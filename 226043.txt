1. git init -го иницијализираме директориумот во кој што сме во моментот
	да ни биде директориумот во кој ќе ги следиме сите промени што се направени
	врз база на проекот со помош на GIT.

2.git remote add origin <repository_url> - го поврзува нашиот локален дирекотирум
	со нашиот github remote репозиториум
  git status - ги следи и ни ги прикажува промените што се направени во датотеките
	и кои датотеки се untacked, staged или треба да бидат commit-нати

3.git add README.md - ја поставуваме датодеката на сцена пред да и направиме commit
  git commit -m "Initial commit" -Откако ја поставивем датотеката на сцена правиме commit
	со пораката "Initial commit" и ова ги зачувува сите промени што се направени од
	последниот commit до овој во git репозиториумот

4.git fetch origin - ги превеземува сите промени што се направени на 
	оддалечениот репозиториум заедно со сите гранки.

5.git branch ни дозволува да креираме посебна гранка на која ќе работиме без да 
	правиме промени на главната гранка master

6. git checkout <ime_branch> ни дозволува да се префрлиме на гранката што сакаме

7.git push - ги поврзуваме промените што се од локалниот репозиториум
	до нашиот remote репозиториум

8.git merge -ја спојуваме гранката на која што сме направиле промена со главната гранка
	или пак со некоја друга гранка
